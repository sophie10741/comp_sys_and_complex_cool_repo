# Лабораторная работа №5

## Разработка GUI-приложения на Java (Swing)

---

## 1. Тема лабораторной работы

Разработка графического пользовательского интерфейса (GUI) на языке Java с использованием библиотеки Swing. Реализация событийной логики, валидации пользовательского ввода и разделения интерфейса и бизнес-логики.

---

## 2. Цель лабораторной работы

Целью лабораторной работы является:

* освоение принципов построения GUI-приложений на Java;
* изучение библиотеки Swing и её основных компонентов;
* формирование навыков обработки событий пользователя;
* применение знаний ООП, исключений, коллекций и файлов в GUI-приложениях;
* развитие навыков проектирования пользовательских приложений.

---

## 3. Задание на лабораторную работу

Необходимо разработать **настольное GUI-приложение на Java (Swing)**, реализующее полноценную пользовательскую логику, обработку ошибок и сохранение данных.

### Общее описание приложения

Приложение должно представлять собой **учётную систему** (тип выбирается преподавателем или студентом):

* список задач (Task Manager);
* учёт студентов/сотрудников;
* простой личный бюджет;
* каталог товаров;
* журнал заметок.

⚠️ **Тип приложения один для всей группы**, реализация — индивидуальная.

---

## 4. Функциональные требования

### 4.1. Интерфейс

Приложение должно содержать:

* главное окно (`JFrame`);
* минимум **2 панели (`JPanel`)**;
* минимум **3 кнопки (`JButton`)**;
* поля ввода (`JTextField`, `JTextArea`);
* текстовые элементы (`JLabel`);
* диалоговые окна (`JOptionPane`).

Интерфейс должен быть логично структурирован и удобен для пользователя.

---

### 4.2. Работа с данными

* данные вводятся пользователем через GUI;
* данные хранятся в **коллекции (`ArrayList`)**;
* реализована возможность:

  * добавления данных;
  * просмотра данных;
  * удаления или изменения данных.

---

### 4.3. Обработка событий

Обязательно реализовать:

* обработку нажатия кнопок;
* реакцию программы на действия пользователя;
* вывод сообщений об ошибках и результатах операций.

---

### 4.4. Валидация и исключения

Приложение должно:

* проверять корректность пользовательского ввода;
* использовать `try-catch`;
* обрабатывать как минимум:

  * `NumberFormatException`;
  * пользовательскую ошибку (логическую).

---

### 4.5. Работа с файлами

Реализовать **хотя бы один** пункт:

* сохранение данных в файл;
* загрузка данных из файла;
* автоматическое сохранение при закрытии программы.

---

## 5. Архитектурные требования

* использовать **минимум 2 собственных класса**;
* логика приложения должна быть вынесена из GUI;
* запрещено хранить всю логику внутри `ActionListener`;
* код должен быть структурирован и читаем.

---

## 6. Требования к отчёту

Отчёт оформляется в электронном виде и должен содержать:

1. Титульный лист
2. Тему и цель лабораторной работы
3. Описание приложения
4. Скриншоты интерфейса
5. UML / блок-схемы (по желанию)
6. Листинг основного кода (фрагменты)
7. Ответы на контрольные вопросы
8. Вывод

---

## 7. Контрольные вопросы (ответить письменно)

1. Что такое GUI и чем оно отличается от консольного интерфейса?
2. Для чего используется `JFrame`?
3. Что такое событие?
4. Зачем нужны обработчики событий?
5. Почему важно разделять интерфейс и логику?
6. Что такое `ActionListener`?
7. Как получить данные из `JTextField`?
8. Какие исключения могут возникать при вводе данных?
9. Что такое коллекции и зачем они нужны?
10. Как осуществляется работа с файлами в Java?
