# Лабораторная работа №6

## Сборка Java-приложения в JAR и мини-проект GUI

---

## 1. Тема лабораторной работы

Сборка настольного Java-приложения в исполняемый JAR-файл. Доработка GUI-приложения до уровня законченного программного продукта (мини-проект).

---

## 2. Цель лабораторной работы

Целью лабораторной работы является:

* формирование навыков подготовки Java-приложения к распространению;
* освоение процесса сборки исполняемого JAR-файла;
* закрепление навыков разработки GUI-приложений;
* развитие навыков проектирования завершённых программных продуктов;
* подготовка к дальнейшей разработке приложений в Android Studio.

---

## 3. Общая постановка задачи

На основе GUI-приложения, разработанного в лабораторной работе №5, необходимо:

1. Доработать приложение до состояния **мини-проекта**;
2. Привести архитектуру и код в порядок;
3. Реализовать сохранение данных;
4. Собрать приложение в исполняемый JAR-файл;
5. Проверить запуск приложения вне среды разработки.

---

## 4. Требования к мини-проекту

Мини-проект должен представлять собой **законченное настольное приложение**, решающее конкретную пользовательскую задачу.

Допустимые варианты тем (одна тема на всю группу):

* менеджер задач;
* учёт записей (студенты, сотрудники, клиенты);
* каталог товаров;
* личный бюджет;
* журнал заметок.

---

## 5. Функциональные требования

### 5.1. Пользовательский интерфейс

Приложение должно содержать:

* главное окно (`JFrame`);
* минимум **3 логических блока интерфейса** (`JPanel`);
* минимум **4 элемента управления** (`JButton`);
* элементы ввода данных (`JTextField`, `JTextArea`);
* текстовые элементы (`JLabel`);
* диалоговые окна (`JOptionPane`).

Интерфейс должен быть:

* логично структурирован;
* понятен пользователю;
* аккуратно оформлен.

---

### 5.2. Работа с данными

В приложении необходимо реализовать:

* ввод данных пользователем;
* хранение данных в коллекции (`ArrayList`);
* добавление записей;
* удаление или изменение записей;
* отображение текущего состояния данных.

---

### 5.3. Сохранение данных

Обязательно реализовать **сохранение данных** одним из способов:

* запись в текстовый файл;
* загрузка данных при запуске приложения;
* автоматическое сохранение при закрытии.

---

### 5.4. Обработка ошибок

Приложение должно:

* проверять корректность пользовательского ввода;
* использовать конструкцию `try-catch`;
* обрабатывать ошибки ввода чисел;
* предотвращать аварийное завершение программы.

---

## 6. Архитектурные требования

* использовать **минимум 3 собственных класса**;
* класс `Main` содержит только запуск приложения;
* интерфейс и логика должны быть разделены;
* запрещено размещать всю логику в обработчиках событий;
* код должен быть читаемым и структурированным.

---

## 7. Сборка JAR-файла

Студент должен:

1. Настроить `Artifact` в среде разработки;
2. Указать корректный `Main-Class`;
3. Собрать JAR-файл;
4. Запустить приложение командой:

```bash
java -jar app.jar
```

5. Проверить запуск вне IDE.

---

## 8. Проверка работоспособности

Перед сдачей лабораторной работы необходимо:

* проверить все элементы интерфейса;
* протестировать ввод некорректных данных;
* проверить сохранение данных;
* убедиться, что приложение корректно закрывается;
* запустить JAR на другом компьютере (или в другой учётной записи).

---

## 9. Требования к отчёту

Отчёт должен содержать:

1. Титульный лист
2. Тему и цель лабораторной работы
3. Описание мини-проекта
4. Скриншоты интерфейса
5. Структуру проекта (пакеты, классы)
6. Примеры ключевых фрагментов кода
7. Описание процесса сборки JAR
8. Ответы на контрольные вопросы
9. Вывод

---

## 10. Контрольные вопросы

Ответить письменно в отчёте:

1. Что такое JAR-файл?
2. Зачем нужен `MANIFEST.MF`?
3. Что такое `Main-Class`?
4. Почему важно запускать приложение вне IDE?
5. Какие проблемы могут возникнуть при сборке JAR?
6. Почему важно разделять интерфейс и логику?
7. Как реализуется сохранение данных?
8. Какие исключения обрабатываются в приложении?
